# Architecture Decision Records

This folder holds the Architecture Decision Records (ADRs) for Move&Meet. Every significant technical choice gets one. ADRs are numbered, immutable once accepted, and never rewritten in place. To change a past decision, write a new ADR that supersedes it.

## Format

Each ADR follows: Context, Decision, Consequences (Positive, Negative/Trade-offs, Neutral), Alternatives Considered, Implementation Notes, Open Questions.

## Status values

- **Proposed**: drafted by the architect agent, awaiting owner approval
- **Accepted**: approved, in force
- **Deprecated**: still informative but no longer the active decision
- **Superseded by ADR-NNNN**: replaced
